開発指示書：ローカル完結型 家庭内在庫管理アプリ

【最優先指令】

本プロジェクトは外部サーバーや外部API（認証・データ保存）を一切利用しない「完全ローカル完結型」のモバイルアプリとして構築してください。オフライン環境下でも全てのCRUD操作が低遅延かつ正常に動作することを絶対要件とします。

1. プロジェクト概要 & 技術スタック

家庭内のあらゆる物品の在庫数、保管場所、カテゴリー、メモなどを一元管理するモバイルアプリケーション。EAS Buildによる実機配布およびストア配信を見据えたモダンな構成で実装してください。

  • ベースフレームワーク: Expo (create-expo-app@latest による最新安定版)
  • 開発言語: TypeScript (厳格な型定義を適用)
  • データベース: expo-sqlite (最新の非同期APIベース)
  • ORマッパー: drizzle-orm / drizzle-kit (型安全なクエリ、自動マイグレーション運用)
  • ルーティング: Expo Router (ファイルベースルーティング)

2. データモデル (Drizzle ORM スキーマ定義)

データの一貫性を保ち、表記揺れを防ぐため、「場所(locations)」および「カテゴリー(categories)」は独立したリレーションテーブルとして定義し、アイテムテーブルから外部キーで参照してください。

2.1 場所マスター (locations)

カラム名 データ型 (SQLite) 制約・オプション 説明
id TEXT PRIMARY KEY, UUIDまたは一意の文字列 場所の個別一意ID
name TEXT NOT NULL 場所の名前 (例: キッチンの床下収納, 洗面所棚)

2.2 カテゴリーマスター (categories)

カラム名 データ型 (SQLite) 制約・オプション 説明
id TEXT PRIMARY KEY, UUIDまたは一意の文字列 カテゴリーの個別一意ID
name TEXT NOT NULL カテゴリー名 (例: 食品, 日用品, 医療品)

2.3 在庫アイテムテーブル (items)

カラム名 データ型 (SQLite) 制約・オプション 説明
id TEXT PRIMARY KEY, UUIDまたは一意の文字列 アイテムの個別一意ID
name TEXT NOT NULL モノの名前 (例: カレールー, 食器用洗剤)
quantity INTEGER NOT NULL, DEFAULT 0 数量（原則0以上の整数）
location_id TEXT REFERENCES locations(id), NULLABLE 保管場所への外部キー
category_id TEXT REFERENCES categories(id), NULLABLE 所属カテゴリーへの外部キー
memo TEXT NULLABLE 補足テキスト（賞味期限、購入時の注意点など）
updated_at TEXT NOT NULL 最終更新日時（ISO 8601 文字列: YYYY-MM-DDTHH:mm:ss.sssZ）

3. ディレクトリ構造の指示

以下のアーキテクチャに準拠してファイルを配置・生成してください。

src/
├── db/
│   ├── schema.ts      # drizzle-orm による上記3テーブルの定義
│   └── client.ts      # expo-sqlite への接続および Drizzle インスタンス初期化
├── components/        # 再利用可能なUIコンポーネント（数量カウンタ、入力フォームなど）
└── app/               # Expo Router 画面ディレクトリ
    ├── index.tsx      # 【メイン画面】在庫アイテム一覧（検索・フィルタリング・数量直感操作）
    ├── item/
    │   ├── add.tsx    # アイテム新規登録画面
    │   └── [id].tsx  # アイテム詳細・編集画面
    └── settings/
        ├── index.tsx  # 設定トップ（マスター管理への導線）
        ├── locations.tsx  # 場所マスターのCRUD画面
        └── categories.tsx # カテゴリーマスターのCRUD画面

4. 画面別 要件定義 & UI/UX ガイドライン

4.1 メイン画面：在庫アイテム一覧 (app/index.tsx)

  • 即時性: アプリ起動と同時に、ローカルDB内の全アイテムをリスト表示すること。
  • クイックカウンター機能: 詳細画面に遷移せずとも、一覧のリストアイテム上に配置された + / - ボタンをタップするだけで、ダイレクトに quantity の増減処理が働き、DBへ即時保存・画面反映されるUIにすること。数量変更時は updated_at も自動更新する。
  • 検索・フィルタ: 「キーワード検索（インクリメンタルサーチ）」「場所別フィルタ」「カテゴリー別フィルタ」を上部に配置し、目的のアイテムへ即座にアクセスできるようにすること。

4.2 登録・編集画面 (app/item/add.tsx, [id].tsx)

  • 入力項目：名前（必須）、数量（デフォルト0、アップダウンコンポーネント）、場所（セレクトボックス/プルダウン）、カテゴリー（セレクトボックス/プルダウン）、メモ（複数行テキスト）。
  • 場所とカテゴリーのセレクトボックスには、各大方マスターに登録されているデータを動的に結合して選択肢として表示すること。
  • 保存処理時、バリデーション（名前が空でないか、数量が負の数になっていないか）を検証すること。

4.3 マスター管理画面 (app/settings/locations.tsx, categories.tsx)

  • アイテム登録を行う前段階、あるいは運用中に「場所」や「カテゴリー」そのものを追加・編集・削除できるシンプルなリストUIを構築すること。

5. 初期化とマイグレーションの自動化指示

アプリケーション起動時に、SQLiteデータベースファイルが自動作成され、Drizzle ORMによる最新のテーブルスキーマが自動適用（マイグレーション）されるロジックを src/db/client.ts またはルートの _layout.tsx 内に組み込んでください。ユーザーにスキーマ更新を意識させてはなりません。

6. 実装ステップ

  1. src/db/schema.ts に記載されたデータベーススキーマの実装。
  2. expo-sqlite を用いたマイグレーションプロバイダ、およびDB接続クライアントの実装。
  3. 「場所」「カテゴリー」のマスターデータを操作する設定画面の構築（動作確認用データの登録を可能にするため、アイテムより先に実装）。
  4. アイテムの新規登録、詳細・編集、一覧表示（およびインライン数量変更機能）の構築。
